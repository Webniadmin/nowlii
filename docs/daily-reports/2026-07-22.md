### 1. AI voice call → OpenAI Realtime API (speech-to-speech over WebRTC)
The old `speech_to_text → SSE → flutter_tts` pipeline felt laggy and the AI interrupted itself. Replaced the
audio engine with the **OpenAI Realtime API** (`gpt-realtime-mini`) over WebRTC — native server-VAD
turn-taking, real barge-in, low latency, hardware echo cancellation. **Verified working on the physical
phone.** Same call screen/design/features; old pipeline kept as a `_useRealtime` fallback. See `realtime-voice.md`.

- Backend `nowli-ai`: new `POST /api/v1/realtime/token` (mints ephemeral session with the persona + voice +
  server_vad + transcription) and `POST /api/v1/session/turns` (feeds the transcript back so the end-of-call
  **summary + emotion/low-mood insights keep working**).
- Frontend: new `realtime_call_service.dart` (WebRTC engine) wired into `ai_voice.dart`.
- **Calm psychological-companion persona** (`_REALTIME_PERSONA_EN`) + **calm female voice `marin`**. Original
  persona untouched (rollback documented).
- **Connecting…** overlay + the call **timer now starts when Nowlii first speaks** (not at connect).
- Mic button = mute toggle; keyboard button = **type instead of talk** (Nowlii replies in voice).

**Two device-independent bugs found & fixed** (the phone kept crashing):
