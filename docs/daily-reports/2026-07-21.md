3. **Discovered the real deploy architecture.** No git repo on the box; services run as Docker images
   (`fahad1000mir/nowlii-*:dev`). Our repo could rebuild the backend but **nowli-ai had no Dockerfile** ‚Äî
   authored one (`nowli-ai/Dockerfile`, `runtime` stage) + `.dockerignore`, committed.
4. **Deployed both services** via `git archive | ssh tar -x` ‚Üí `docker compose build && up -d`, with the
   current images tagged `:backup-20260721` for rollback. Two gotchas hit + fixed:
   - **CRLF crash-loop** ‚Äî `entrypoint.sh` shipped with `^M`; `git archive` on Windows re-injects CRLF via
     `core.autocrlf=true`. Fix = deploy with `git -c core.autocrlf=false archive ‚Ä¶` (added a `.gitattributes`
     too). The repo blob itself is correct LF.
   - **400 on every request** ‚Äî box `.env` had no `ALLOWED_HOSTS`; added it + `CSRF_TRUSTED_ORIGINS`.
5. **Built the debug APK** against AWS and put it on the Desktop for file-transfer install (no phone
   tether needed).
6. **Debugged on-device login failures.** Confirmed (via packet capture on the box) the phone reaches AWS
   fine. Root cause of "request failed": (a) Google login returned **503** because the prod `.env` lacked
   `SOCIAL_AUTH_GOOGLE_CLIENT_ID` ‚Üí **added it, restarted, now 401‚Üíworks on device**; (b) the app's
   `api_service.dart` masked every server error as "Request failed" ‚Üí **rewrote it** to surface the real
   status/message + network exception (debug build) and rebuilt the APK.
7. **AI voice = silence** ‚Üí nowli-ai logs show `openai.RateLimitError 429 insufficient_quota`. The shared
   OpenAI key's account is **out of credits** ‚Äî affects local too. Pavle is adding OpenAI billing.
8. **Full live audit** (see `deploy-aws.md`): every endpoint probed with a minted superuser JWT. Core app
   is healthy; only AI features (voice, insights, subtask-gen) are down, all via the one OpenAI quota.
