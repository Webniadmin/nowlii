# Daily Reports

One report per working day, written at end of day. File name: `YYYY-MM-DD.md`
(e.g. `2026-07-06.md`). These are a running history; the user archives them out of the
project periodically, so they are not meant to be kept forever.

Each report follows the template below.

---

## Template

```markdown
# Daily Report — YYYY-MM-DD

## Done today
- <what was actually completed>

## Changes made
- <files / areas changed, endpoints added, config touched — enough to trace later>

## Still open
- <started-but-unfinished, blockers, anything punted to future-checklist.md>

## Recommended next start
- <where to pick up tomorrow, and why>
```
