  `call_summary_screen.dart` card defaults + backend `_SUMMARY_FALLBACKS` (en/de/es).

### Profanity / content filter (nowli-ai)
- Hybrid moderation in `chat_stream`: local `_PROFANITY_WORDS` (EN+SR) + OpenAI
  `omni-moderation-latest` (harassment/hate/sexual). **Deliberately excludes self-harm/distress**
  (wellness app). On block → `warning` SSE frame → app shows a notice + speaks a gentle TTS warning;
  the model + turn are skipped. Verified: clean→reply, profanity→blocked, distress→full support.

### Latency + architecture change (fluid conversation)
- Root cause of lag: **per-message emotion detection** (a gpt-4o call before every reply, ~1.5–4s).
  Moved emotion extraction to **end-of-call over the whole transcript** (client's idea: N calls → 1).
- chat-stream now runs only moderation then streams the reply → **first word ~0.9–1.4s** (httpx-
  measured; was ~2.5–8s). Old per-message code **commented, not deleted**.
- `_compute_top_emotions_from_transcript()` (env `EMOTION_MODEL`, default `gpt-4o-mini`) feeds
  **call-insights** (Top Emotions + When-feeling-low). The **summary** GPT call now also returns
  `top_emotions`; a new **"Emotions in this chat"** section shows them on the call-summary screen.

### AI persona
- Since emotion is now always "neutral", only the "neutral" persona runs. Rewrote it (en/de/es)
  from a casual "close friend / don't sound like a helper" into an **emotionally-intelligent warm
  wellness companion**. Emotion-specific templates are now dead (kept for reference). Verified live.

### Barge-in (interrupt the AI)
- Implemented client-side (mic open during TTS; real user phrase stops the AI). Emulator testing
  showed it **breaks without hardware echo cancellation** (mic hears the AI's own TTS → self-
  interrupt loop + STT dies + summary 20%×5). Disabled, then **re-enabled at client request** — with
  the hard requirement of **headphones on the emulator or a real phone (AEC)**. Added a
  **tap-to-interrupt** fallback (tap the mic while the AI talks) that works even with speakers.
