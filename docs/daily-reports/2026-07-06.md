## Call-screen UI fixes — two rounds (`294793a`, `069e909`)

Fixed inherited in-call visual issues on the live call screen (`lib/screen/ai_call/ai_voice.dart`);
logged as new register entries **TD-017…TD-021**. Scope-limited, no new deps, UI/UX unchanged
beyond the fixes.

- **TD-017 — mic icon flicker.** The icon tracked the raw `speech_to_text` lifecycle
  (`_isListening` flips on every pause). Round 1 added a debounced `_micActive`; round 2 (after
  on-device testing) switched it to **voice activity** via `_speech.listen`'s `onSoundLevelChange`
  (+ `onResult`): red while the user actually speaks, off ~1s after they stop (one non-resetting
  `Timer`). Recognition/restart logic untouched. `_micSpeakingThreshold` may need per-device tuning.
- **TD-018 — timer shifted the layout.** Proportional Wosker digits changed the timer's width.
  A fixed-width `SizedBox` (round 1) still let the seconds digits shift, so round 2 renders
  **per-digit fixed-width slots** (`_measureDigitWidth()` via `TextPainter`, `_fixedWidthTime()`).
  Font/size/colors unchanged.
- **TD-019 — last minute turned orange.** Removed the orange from `_backgroundColor` (round 1)
  **and** `_timerColor` (round 2 — the orange also washed out the digits); dropped the now-unused
  `_isTimeWarningActive`. Nothing recolors on the last minute; only the notice card signals it.
- **TD-020 — final-10s fullscreen overlay.** Replaced with `_buildCountdownNotice()` on the shared
  `_noticeCard` (counts 10 → 1); old overlay commented out (preserve-not-delete).
- **TD-021 — placeholder heading + summary error.** Heading `Answer emails` → neutral companion
  `Let's talk` / `All done`; and the call summary now falls back to its default cards instead of a
  `No session ID provided` error when the AI session is missing (`call_summary_screen.dart`).
- **Verified:** `flutter analyze` 0 errors, no new warnings (`ai_voice.dart` 69 → 68). Docs

**Top Emotions** (happy/motivated/angry/tired/sad) and **When feeling low, you often say…** —

- **Plan (for 2026-07-07):** reconcile the 5 categories → capture the two nowli-ai analyses at
  call end (session still in memory) → persist per-user in Django (extend `Apps/voice_calls` end) →
  aggregate in `Apps/insights` → add frontend model fields + the two UI sections. Full report +
  touch list in **`docs/insights-emotions.md`**; tracked in `future-checklist.md` (P3),
  `daily-checklist.md` (next up), `project-status.md` (unconsumed analytics endpoints).
