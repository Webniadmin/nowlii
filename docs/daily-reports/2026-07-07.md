### P0 — "AI doesn't respond during the call" — root cause + code robustness
- **Root cause = environment, confirmed with evidence:** the STT→`_sendMessageToAi`→`chatStream`→TTS
  wiring is correct; AVD `Medium_Phone.avd` has `hw.audioInput=yes` + a Google Play image; Windows mic
  privacy = Allow. The earlier failure was the emulator not routing host audio to the guest mic
  (`error_speech_timeout`). Once the mic was enabled (Extended controls → Microphone → "Virtual
  microphone uses host audio input"), the AI responded normally.
- **Code fix (`ai_voice.dart`):** added a one-time "I can't hear you — check that your microphone is
  on." hint after repeated speech-timeout/no-audio errors (was silently looping); resets on the first
  recognized words. 2 state fields + `onResult` reset + `onError` branch.

### P3 — "Could not load summary" — fixed
`call_summary_screen.dart` `_loadSummary`: a null summary (nowli-ai 422 for an empty session / any
failure) now **falls back to the default insight cards** instead of the error screen; `_errorMessage`
is never set. Unifies with the missing-sessionId branch (TD-021). Verified live: call 14 loaded a real
summary (200); the fallback covers the empty-session case.
